# Техническое задание (ТЗ) по теме "Многофункциональный центр (МФЦ)"

## Роли пользователей

Система подразумевает наличие трех типов пользователей:

1. **Неавторизованный пользователь (Гость)**:
   - **Возможности**: 
     * Просмотр списка всех отделений
     * Просмотр детальной информации об отделении (включая фото и часы работы)
   - **Права**: Только чтение информации. Не может создавать записи, редактировать или удалять данные.
   - **Ограничения**: Не видит кнопку "Записаться на услугу". При попытке прямого перехода по URL записи перенаправляется на страницу входа.

2. **Авторизованный пользователь (Клиент)**:
   - **Возможности**: 
     * Просмотр списка отделений
     * Просмотр детальной информации об отделении (включая фото и часы работы)
     * Запись на услугу в выбранном отделении
   - **Права**: Только чтение информации и создание новых записей на прием. Не может редактировать или удалять существующие данные (отделения, сотрудников).
   - **Ограничения**: Не может записаться в неактивное отделение.

3. **Администратор (Admin)**:
   - **Возможности**:
     * Просмотр списка отделений
     * Просмотр детальной информации об отделении
     * Создание нового отделения
     * Редактирование существующего отделения
     * Удаление отделения
     * Просмотр всех записей пользователей (в админке)
   - **Права**: Полный доступ к управлению объектами (CRUD - Create, Read, Update, Delete) для отделений МФЦ и просмотр записей.
   - **Ограничения**: Не может записываться на услуги (это функция клиентов).

## Бизнес-логика

1. **Управление отделениями (Администратор)**:
   - **Создание**: Администратор может добавить новое отделение МФЦ, заполнив форму с названием, адресом, телефоном, email, графиком работы и прикрепив фото.
   - **Редактирование**: Администратор может изменять любую информацию о существующем отделении, включая его статус (активно/неактивно) и загружать новое фото.
   - **Удаление**: Администратор может удалить отделение. Перед удалением система запрашивает подтверждение, чтобы избежать случайных действий.
   - **Просмотр**: Администратор видит все отделения в списке и может перейти к просмотру детальной информации о каждом.

2. **Запись на услуги (Авторизованный пользователь)**:
   - На странице детальной информации об отделении пользователь видит кнопку "Записаться на услугу" только если отделение активно.
   - При нажатии на кнопку открывается форма для записи.
   - Пользователь выбирает услугу из списка доступных в этом отделении.
   - Пользователь указывает дату и время визита.
   - После заполнения всех обязательных полей и отправки формы создается новая запись в системе.
   - После успешной отправки пользователь перенаправляется обратно на страницу отделения и видит сообщение об успешной записи.
   - Новая запись получает статус "Ожидает" (PENDING).

## Валидация

1. **Создание/Редактирование отделения (Форма `branch_form.html`)**:
   - **Название**: Обязательное поле. Минимальная длина — 3 символа. Максимальная — 200 символов.
   - **Адрес**: Обязательное поле. Минимальная длина — 10 символов.
   - **Телефон**: Обязательное поле.
   - **Email**: Обязательное поле. Проверяется на корректность формата (наличие `@` и домена).
   - **График работы**: Обязательное поле. Минимальная длина — 10 символов.
   - **Уникальность email**: При создании или редактировании отделения проверяется, что указанный email не используется другим отделением.
   - **Фото**: Максимальный размер файла — 5 МБ.

2. **Запись на услугу**:
   - **Авторизация**: Только авторизованные пользователи могут создавать записи.
   - **Активность отделения**: Запись возможна только в активные отделения (`is_active=True`). Если отделение неактивно, при попытке прямого доступа к форме записи выдается сообщение об ошибке.
   - **Услуга**: Обязательное поле. Пользователь должен выбрать одну услугу из списка.
   - **Дата**: Обязательное поле. Не может быть в прошлом.
   - **Время**: Обязательное поле. Не может быть выбрано отличное от рабочих часов.
